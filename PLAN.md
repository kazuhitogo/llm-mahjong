# 実装プラン（残タスクのみ）

## Phase 7: Viewer 3 カラムレイアウト

**目的**: ステップ送りで説明文の高さが変動 → 直下の卓が上下にズレる問題を解消する。
卓を中央固定にし、可変高さの要素（説明・推論・プロンプト）を卓と別カラムへ隔離する。

**最終レイアウト**:

```
左カラム(固定幅 ~240px)   中央              右カラム(固定幅 ~300px)
─────────────────────  ──────────────  ─────────────────────
タイトル / ログ読込 / seed   麻雀卓 720px       イベント説明
局タブ(縦並び/折返し)        (位置不動)         推論(think 時)
再生コントローラー                            入力プロンプト(縦スクロール)
 ⏮◀▶⏭ / スライダー / n/N
POV / 全開示
```

**タスク**:
1. `App.tsx`: 上部緑ヘッダーバーを廃止し、中身（タイトル・ログ読込・seed・POV・全開示）を左カラム上部へ移す
2. `App.tsx`: 局タブを左カラムへ移動（縦並びまたは折り返し）
3. `App.tsx`: 再生コントローラー（ボタン・スライダー・カウンタ）を左カラムへ
4. `App.tsx`: イベント説明・推論・プロンプトを右カラムへ。右カラムは縦スクロール（卓は不動）
5. `App.tsx`: 全体を 3 カラム flex に。一定幅未満で縦積み（左→中央→右）にフォールバック
6. `docs/viewer.md` のコンポーネント構成・UI コントロール節を実装に合わせて整合

**完了条件**: `pnpm typecheck` 通過。`pnpm viewer` でステップ送り時に卓が一切動かないことを目視確認。
